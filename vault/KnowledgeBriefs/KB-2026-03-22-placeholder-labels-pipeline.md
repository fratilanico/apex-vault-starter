# Pipeline-Generated Content Must Have Real Labels

## Discovery

The lesson schema generator used "Afbeelding 1", "Afbeelding 2", "Afbeelding 3" as exercise option labels. All 3,723 goals had these meaningless placeholders. A teacher would immediately reject this as unusable — the options in an exercise should describe what the student is choosing, not just number them.

"Afbeelding" means "Image" in Dutch. So students saw three identical labels saying "Image 1", "Image 2", "Image 3" next to pictograms, with no indication of what each pictogram represents.

## Root Cause

The schema generator pipeline had a fallback label: `f"Afbeelding {i + 1}"`. This was meant as a development placeholder, but no check was ever added to verify that real labels replaced placeholders before data was committed to production.

## Rule (permanent)

**Every user-facing text generated by a pipeline MUST be a real, meaningful label in the target language.** Numbered placeholders ("Item 1", "Option A", "Afbeelding 3") are development artifacts, not content. They must never reach production.

## Detection Pattern

```python
PLACEHOLDER_PATTERNS = [
    r"^Afbeelding \d+",       # Dutch placeholder
    r"^Image \d+",            # English placeholder
    r"^Item \d+",             # Generic numbered
    r"^Option [A-Z]$",        # Lettered option
    r"^Lorem",                # Lorem ipsum
    r"^TODO",                 # Developer note
    r"^test",                 # Test data
    r"^placeholder",          # Explicit placeholder
    r"^untitled",             # Default title
    r"^\[.*\]$",              # Bracketed placeholder [name]
]

import re

def audit_labels(schemas: list[dict]) -> list[str]:
    """Find placeholder labels in generated content."""
    violations = []
    for schema in schemas:
        for ex in schema.get("exercises", []):
            for opt in ex.get("options", []):
                text = opt.get("text", "")
                for pattern in PLACEHOLDER_PATTERNS:
                    if re.match(pattern, text, re.IGNORECASE):
                        violations.append(
                            f"Placeholder found: '{text}' in goal {schema.get('goal_id')}"
                        )
    return violations
```

## Correct Pattern

Maintain a label mapping that provides real Dutch words for every known pictogram ID:

```typescript
const PICTOGRAM_LABELS: Record<number, string> = {
  2345: "Tellen",        // Counting
  6789: "Boek",          // Book
  1234: "Gevoel",        // Feeling
  // ... complete mapping
};

function getLabel(picId: number, index: number): string {
  const real = PICTOGRAM_LABELS[picId];
  if (!real) {
    console.warn(`Missing label for pictogram ${picId} — add to PICTOGRAM_LABELS`);
    return `afbeelding ${index + 1}`;  // Fallback — will be caught by QA
  }
  return real;
}
```

## 4D QA Plane

This is a **Plane 2: Data Integrity** issue (check P2-06: No placeholder/test data in production).

## Impact

- 3,723 goals had meaningless labels
- Teachers could not use exercises meaningfully
- Students had no context for their answer choices
- Full regeneration required after label mapping was built

## Prevention

- Run placeholder audit in CI pipeline
- Plane 2 check P2-06 at every wave:checkpoint and wave:complete
- Pipeline must fail-fast on any placeholder match, not silently continue
